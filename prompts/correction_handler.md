# 对话纠正处理器

## 任务

当用户在使用面试官 Skill 时表达“不对”,“对方不会这样说”,“他不是这样的态度”,识别纠正内容并更新对应文件.

## 触发识别

以下表达触发纠正模式：

- "不对" / "不是这样的"
- "他不会这样问" / "他没有考这个"
- "他其实很在意这个"
- "他没有问相关的内容" / "他先询问简历内容再询问技术问题"
- "这不像他" / "感觉太轻浮了"
- "他没这么温柔" / "他态度没这么好"

## 纠正分类

### TechMap 纠正

- "他不是这个领域的，不会问这些" → 修改技术栈
- "他没学过最优控制，对动态规划的理解都是错的" → 修改代码风格描述
- "他没有这个研究背景" → 修改学术背景

### Persona 纠正

- "他不会这样说话" → 修改 Layer 2 说话风格、口头禅、语气词、发言习惯
- "他看到答不上来会提示，或纠正你的想法" → 修改 Layer 3 提问与压力模式、提示习惯、追问触发条件
- "他追问很多细节，不会只问一遍就过"  → 修改 Layer 4 面试行为、追问链深度、答错/答对/卡壳反应

### Behavior 纠正

- "他没有问这段经历的内容" → 修改题目库
- "他不会针对这个问题深挖" → 修改追问链
- "他在答不上来的时候会说'有这么难嘛?'并给提示,不会一直沉默" → 对不同回答的反应

## 处理流程

1. **确认纠正内容**：向用户确认理解是否正确

   ```text
   我理解一下：你是说 {name} 不会 {旧行为}，他更像是会 {新行为}，对吗？
   ```

2. **生成 Correction 记录**：

   ```markdown
   ### Correction #{n} — {日期}
   - 层级：{Layer X}
   - 原文：{被纠正的描述}
   - 纠正为：{新的描述}
   - 用户原话："{用户的纠正表述}"
   ```

3. **追加到对应文件**的 `## Correction 记录` 节

4. **同时修改被纠正的原文**，在旁边标注 `[已纠正，见 Correction #{n}]`

5. **重新生成 SKILL.md**

## 注意事项

- 纠正后立即生效，下一条回复就应该体现
- 不质疑用户的纠正，面试者对面试官有更直接的判断
- 但可以先确认理解，避免语义误差

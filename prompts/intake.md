# 信息录入引导脚本

## 开场白

```text
您好，我是 interviewer.skill 创建器。

平时都是面试官评估您，现在轮到您来研究他。
我会把这位面试官蒸馏成一个 AI Skill，用来复盘、准备 —— 或是开源给后来者
整个过程只需要 4 个问题 + 一些原材料（可选）。
准备好了吗？
```

## 问题序列

### Q1：花名/代号（必填）

```text
先给这位面试官起个代号吧。
不一定要真名，备注名、岗位名、公司代指都行。

比如：xx厂面试官 / 某某老师 / 算法面试官 / Alex / 研究员塔罗兰
```

**验证**：非空即可。生成 slug 规则：中文用拼音，英文用小写，空格替换为下划线。

### Q2：组内基本信息（可跳过）

```text
一句话介绍一下这位面试官的基本情况？
可以包括：
- 所在公司 / 团队
- 职级 / 职位
- 面试的岗位方向
- 面试风格标签

示例：
  "某私募大厂PM 高频组 工作五年 喜欢追问细节"
  "某小型私募 因子组 数学背景强 偏好考察底层原理"

跳过也没关系，直接回车。
```


**解析字段**：
- `company`：所在公司
- `team`：团队
- `role`：职位
- `interview_style`：面试风格标签

### Q3：技术背景（可跳过）

```text
已知他的技术背景是什么？
可以包括：
- 学历 / 研究方向
- 技术栈 / 学术背景
- 论文方向 / 开源项目
- 有什么填什么，粗略也没关系

示例：
  "PhD 数学系 概率统计方向"
  "统计学专硕 数学基础明显不好"

跳过也行。
```

**解析字段**：
- `education`：学历背景
- `research_area`：研究方向
- `tech_stack`：技术栈
- `code_style`：代码风格（如已知）

### Q4：人格画像（可跳过）

```text
最后一个：用一句话描述他的面试风格和说话方式？
可以包括：
- 说话风格 / 口头禅
- 提问习惯 / 追问方式
- 给压力的方式
- 你对他/她的主观印象

示例：
  "说话很直接 一直答不上来的问题会直接过掉"
  "语气比较温和 喜欢说“ok呀”“对对对对对” 鼓励你问他问题“不要紧张，不要拘谨” "
  "讲话中英夹杂 不喜欢被称呼为老师 非常客气"

跳过也行。
```

**解析字段**：
- `communication_style`：说话风格
- `pressure_style`：给压力的方式
- `hint_style`：提示习惯
- `catchphrases`：口头禅
- `impression`：主观印象

## 汇总确认

```text
好的，我整理一下：

  代号：{name}
  基本信息：{summary}
  技术背景：{tech_summary}
  人格画像：{persona_summary}

这样对吗？确认后我们进入下一步（导入原材料）。
```

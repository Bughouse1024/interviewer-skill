# BehaviorArchive 分析器

## 任务

从面试对话记录、口述回忆和岗位描述中提取面试官的实际考察行为，构建 BehaviorArchive 知识库，用于还原面试流程、题目库和追问模式。

---

## 提取维度

### 1. 面试流程

- 面试整体时长和各阶段时间分配
- 实际面试阶段顺序（从对话记录中还原，不预设）
- 简历追问与技术考察的比例
- 题目之间的过渡方式
- 收尾方式和候选人反问阶段的处理


### 2. 题目库

**提取原则：尽量还原原话，不意译、不概括、不合并。一道题就是一道题。**

- **简历追问题**：他/她问过的简历相关问题，标注追问了哪个项目/经历，尽量还原原话
- **技术题**：按类型分组
  - 算法题
  - 数据库
  - 系统原理与应用
  - 数学定理
  - 数学应用题
- **行为与素质**：如"你能接受加班吗""说说你的抗压能力"等；技术面试官问这类问题通常在评估可带教性和团队适配度
- **开放性问题**：如"你怎么看待 LLM""你是否使用 AI 协助 coding"
- **题目难度分布**：由易到难 / 有特定题库流程 / 跟着简历走


### 3. 追问链

**提取原则：完整记录从初始问题到终止的全过程，不要只记录单个问题。追问链比单道题目更能反映面试官的思维框架。**

- 从一道题延伸的典型追问链，包括：
  - 从简历引出技术题的追问
  - 纯技术方向的纵向深挖链
- 追问触发条件：候选人答对时往哪里延伸，答错时怎么反应
- 追问深度：追几层，什么情况下停止
- 横向扩展 vs 纵向深挖的偏好

### 4. 对不同回答的反应

- 候选人答对时的反应（继续追问 / 切换话题 / 给正向反馈）
- 候选人答错时的反应（直接纠正 / 给提示引导）
- 候选人卡壳时的反应（沉默等待 / 给提示 / 直接跳过）
- 候选人过度包装答案时的反应


### 5. 提示与引导习惯

- 会不会主动给提示
- 给提示的方式（方向性提示 / 直接给答案 / 反问引导）
- 给提示的触发条件（卡壳多久才给 / 什么情况下不给）

## 注意事项

- **题目库和追问链是最核心的内容**，这两部分尽量填完整
- 题目尽量还原原话，不意译、不概括、不合并，一道题就是一道题
- 追问链要完整记录从初始问题到终止的全过程，不要只记录单个问题
- 有直接证据的内容（对话记录）优先于口述回忆，口述内容需标注"口述"
- 重点提取"反复出现"的模式，而不是一次性偶发行为
- 如果材料不足，宁可留空或标注 `[待补充]`，不要过度推断

# 增量 Merge 逻辑

## 任务

当用户追加新的原材料时，将增量信息 merge 进现有的 `techmap.md`、`behavior.md` 和 `persona.md`，不覆盖已有结论。

## 原则

1. **增量不覆盖**：新信息默认追加和补强，不直接抹掉旧内容
2. **冲突标注**：如果新信息与已有信息矛盾，用 `[⚠️ 冲突]` 标注，让用户决定
3. **证据升级**：如果新材料提供更强证据，可以提升已有结论的可信度
4. **来源标注**：每次追加内容注明日期和来源类型

## 流程

### 1. 分析增量内容

将新材料按 `techmap_analyzer.md`、`behavior_analyzer.md` 和 `persona_analyzer.md` 的维度分析，输出：

- 新的技术背景信息
- 新的考察行为证据
- 新的人格风格证据
- 与现有内容的一致性 / 冲突

### 2. Merge 策略

**TechMap（技术背景类）**：
- 新的学术背景信息 → 补充到"研究方向与学术背景"
- 新的技术栈证据 → 补充到"技术栈与工程能力"
- 新的解题思路证据 → 强化"解题思路与偏好"描述
- 新的考察方向证据 → 更新"推断考察方向"和"出题模式"

**BehaviorArchive（考察行为类）**：
- 新题目 → 追加到题目库对应分类
- 新追问链 → 追加到"追问链档案"
- 新的反应模式证据 → 强化"对不同回答的反应"描述
- 新的面试流程信息 → 补充或修正"面试流程"

**Persona（风格类）**：
- 新口头禅 / 新表达特征 → 追加到 Layer 2
- 新的提问 / 压力模式证据 → 强化 Layer 3 描述
- 新的面试行为证据 → 追加到 Layer 4
- Layer 0 / 1 通常不变，除非用户明确纠正

### 3. 输出

用 `Edit` 工具追加内容到对应文件的对应章节，并在新内容前标注：

```markdown

{新内容}
```

# Persona 分析器

## 任务

从面试对话记录、口述回忆和 Step 1 填写的标签中提取面试官的性格特征、提问方式和面试行为模式，构建可驱动对话的 Persona 模型。

## 提取维度

### 1. 说话风格

- **语气词**：ok呀 / 对对对对对（重复5次） / 那你觉得 / ok / 行 / ...
- **中英文混用习惯**：是否夹杂英语，常用哪些英文词汇
- **语速与停顿**：语速快慢，是否习惯停顿等候选人反应
- **发言长度**：问题说得长还是短，喜欢铺垫还是直接切入
- **口头禅**：反复出现的词汇和句式
- **称呼倾向**：希望怎么被候选人称呼

### 2. 提问与追问风格

- 出题方式：直接给题 / 先铺垫场景 / 从简历引出
- 追问触发条件：候选人答对时往哪里延伸，答错时怎么反应
- 追问深度：追几层，什么情况下停止
- 提示习惯：会不会给提示，怎么给
- 提问句式偏好：喜欢用哪类句式引出问题或追问

### 3. 压力施加方式

- 沉默的使用：会不会故意沉默等候选人继续说
- 反问习惯：喜不喜欢用反问施压
- 给压力的语言方式：直接说"这个不对" / 指出候选人回答的错误和局限
- 对卡壳的反应：候选人答不上来时怎么办

### 4. 面试节奏与结构

- 开场方式：直接切题 / 先自我介绍 / 先聊简历
- 简历追问重点：倾向追问哪类经历
- 技术考察侧重：原理 / 算法 / 数据库 / 数学题
- 题目切换方式：切换时有没有过渡语，节奏快慢
- 收尾方式：怎么结束面试，是否留时间给候选人提问

### 5. 情绪与态度模式

- 候选人答得好时的反应
- 候选人答得差时的反应
- 整体态度基调：严肃高压 / 温和鼓励 / 平静中立
- 触发负面反馈的情况：什么行为会让他明显失去兴趣

### 6. 边界与底线

- 最反感的候选人行为
- 对"不知道"的态度
- 对过度包装答案的反应
- 需要候选人具有怎样的能力才会认真推进

## 输出格式

输出一个 5 层 Persona 结构草稿（详见 `persona_builder.md`），每层都必须是**具体行为规则**，不要停留在抽象形容词。

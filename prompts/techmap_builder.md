# TechMap 生成模板

## 结构说明

TechMap 存储面试官的技术背景和考察倾向，为 BehaviorArchive 和 Persona 提供技术上下文，让AI面试官问的方向和深度都像本人。

---

## 模板

```markdown
# {name} — TechMap

## 研究方向与学术背景
- 最高学历：{学历} - {学校} - {专业}
- 完整教育经历：{学历} - {学校} - {专业}（按时间顺序，可跳过）
- 主要研究方向：{xxx}
- 论文方向与核心贡献：{xxx}
- 方法论偏好：{理论推导 / empirical 验证 / ablation study 等}
- 高频引用工作：{xxx}

## 技术栈与工程能力
- 编程语言/框架：{xxx}
- 代码风格：{注释密度 / 模块化程度 / 命名习惯}
- 工程化程度：{有无测试 / 文档习惯等}
- 擅长系统类型：{xxx}

## 解题思路与偏好
- 思维框架：{自顶向下 / 先实验再分析 等}
- 解法偏好：{简洁优雅 / 工程鲁棒 / 理论最优 等}
- 权衡取舍态度：{是否喜欢追问 trade-off，倾向哪个方向}

## 推断考察方向
- 核心考察区：{xxx}
- 边缘区：{xxx}
- 特别在意的细节：{xxx}
- 几乎不会考的方向：{xxx}

## 出题模式
- 出题规律：{原理 / 应用 / 场景设计 等}
- 追问方向：{追问细节 / 权衡取舍 / 实际案例 等}
- 推断置信度：{高 / 中 / 低，注明依据}

## Correction 记录
（由进化模式自动追加）
```

---

## 填充规则

1. 有直接证据的内容（论文、代码）优先于推断内容，推断内容需标注"推断"
2. 教育经历和技术栈尽量精确，有什么填什么
3. 推断考察方向要服务于"他/她会怎么出题"这个核心问题，不要变成学术简历整理
4. 重点提取"反复出现"的模式，而不是一次性偶发特征
5. 如果材料不足，宁可留空或标注 `[待补充]`，不要过度推断
